Discuz论坛搬迁记录

备份Discuz原站数据

在删除Discuz原站数据之前，我们须对其进行全面的备份，必要的数据包括：
原Discuz源代码、配置文件以及图片等静态资源
原Discuz数据库中的所有表

备份原Discuz的源代码等资源

只需要完整地复制原程序目录即可。

备份原Discuz数据库中的所有表

一般来说有两种操作路径：
后台->站长->数据库->备份->Discuz!和UCenter数据
后台->站长->数据库->备份->自定义备份

对于后者，我们可以手动选择所有数据表进行保存。

但对于前者，系统是不会对数据库中的common类型的表进行备份的。
遇到仅备份Discuz!和UCenter数据的情况，请务必保留原Discuz系统的安装包，因为我们需要使用*{root_dir}/install/index.php*来恢复common表。（index.php是Discuz安装程序的入口，在论坛构建完成时后台为了系统安全性会要求用户将其删除）

备份下来的表数据会以.sql文件的形式保存在*{root_dir}/data/backup_xxxxxx*中。

配置新的服务器环境

在裸CentOS系统中，我们需要安装并配置好几个必要的软件，它们包括：
服务器软件Nginx及其依赖
解析软件PHP及其他一系列的支持模块
数据库软件MySQL

更新系统工具

  • 在裸系统的前提下，为了保证后续的编译安装工作能顺利进行，我们得做一些预备工作：
    • 更新yum源
    • 利用yum来安装一些必备工具，如GCC等

# 将新的yum源文件下载到yum.repos.d文件夹内
wget -P /etc/yum.repos.d http://mirrors.163.com/.help/CentOS6-Base-163.repo

# 备份旧的yum源文件
mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.bak

# 用新的yum源文件替代旧的yum源文件
mv /etc/yum.repos.d/CentOS6-Base-163.repo /etc/yum.repos.d/CentOS-Base.repo

# 缓存包信息，提升后续的查找速度
yum makecache

# 安装GCC、G++等工具
yum install gcc gcc-c++ libstdc++-devel
